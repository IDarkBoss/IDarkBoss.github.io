PuTTY

rcm 的开发数据库是在内网 192.168.8.116上，不能直接连接，需要使用 SSH Tunnel 方式连接。可以使用 PuTTY 或者 Secure crt 来完成，配置方式和 ComStar 前中台连接开发数据库类似。

SSH Tunnel 的基本原理

基本原理是，以 172.19.223.218为跳板，该服务器有公网 IP，也同时有内网 IP 192.168.8.1。建立连接后，同时建立本地端口 20051 到 192.168.8.116:1521 （数据库的监听端口）的映射，该映射复用已有的到 172.19.223.218的连接。在需要连接数据库的时候，只要连接本地端口 20051，在 172.19.223.218的连接已经建立的情况下，会发送到该服务器并转发到目标服务器端口 192.168.8.116:1521，如果与 172.19.223.218的连接断开，那么与内网的其他连接也全部断开。具体可以参考 SSH Tunnel 相关文档。

以上配置完成后，连接 PuTTY 或 Secure crt，然后启动 rcm 服务即可成功连接到数据库。

注意：配置的本地端口 20051是可以改变的，只要与本地的 application.properties中配置的 jdbc 的端口一致即可。可以改变。但只建议连开发环境，且因为 TIS 轮询机制的存在， 当多人同时起 TIS 时， 可能会造成收不到消息的现象。

PuTTY 配置 SSH Tunnel

先新建一个 Session，在 Session 页面，填写“Host Name”为 172.19.223.218，端口 22，用户名密码请询问运维组人员。

然后，在该 Session 中，“putty configuration”->“category”->“SSH”->“Tunnels”中，添加 Tunnels。最后，返回 Session 页面，并返回 Session 选项点击 Save，保存配置到当前 Session 中。

本地 Zookeeper 服务

本服务需要在启动 rcm 和 tis 服务前先启动。

  1. 首先下载 Zookeeper，并解压到本地

  2. 进入 conf 目录修改 zoo_sample.cfg 文件名为 zoo.cfg

  3. 打开 zoo.cfg 修改 data 目录为（新建 data 目录）：dataDir=D:\\zookeeper\\data （路径根据自己的路径填写）

  4. 创建 zkStart.bat 文件，文件内容为："Your Path"\bin\zkServer.cmd

  5. 双击打开 zkStart.bat 运行 zookeeper（必须配置 JAVA_HOME 环境变量）

maven 配置

maven 主要需要配置仓库的地址，配置文件在{user.home}/.m2/setting.xml。直接用 SVN 上的setting.xml（地址在：http://192.168.7.201:18088/Rcs/rcm/settings.xml）替换原有的setting.xml。

配置完 maven 后，在命令行工具下，进入 rcm 项目的根目录，运行mvn install -D maven.test.skip=true指令可以进行项目构建，会需要一段时间，耐心等待。首次构建，maven 会下载相关的 jar 包和 plugin 到本地。如看到了 maven 的构建提示，说明 maven 配置无误。

注意 tis 下的tis-base-info/pom.xml中的版本号需要为 1.9.2-SNAPSHOT（以最新为准），否则编译时可能会找不到 jar 包。

Sencha cmd 6 配置

Sencha cmd 6 不需要配置即可使用，但首次使用的时候，需要构建整个 web 工程。

  • 进入rcm-server/src/main/webapp/，运行sencha app build classic。
  • 进入tis-server/src/main/webapp/，运行sencha app build classic。
